## K-평균 군집화
- K-평균 군집화는 데이터를 입력받아 소수의 그룹으로 묶는 알고리즘이다.
- 레이블이 없는 데이터를 입력받아 각 데이터에 레이블을 할당해서 군집화를 수행하는데, 학습 과정은 다음과 같다.

<br/>

1. 중심점 선택: 랜덤하게 초기 중심점을 선택
2. 클러스터 할당: 
    - K개의 중심점과 각각의 개별 데이터 간의 거리를 측정한 후, 가장 가까운 중심점을 기준으로 데이터를 할당한다.
    - 아 과정을 통해 클러스터가 구성된다(이때 클러스터링은 데이터를 하나 혹은 둘 이상의 덩어리로 묶는 과정이며, 클러스터는 덩어리 자체를 의미한다.)
3. 새로운 중심점: 클러스터마다 새로운 중심점을 계산한다. 
4. 범위 확인: 
    - 선택된 중심점에 더 이상의 변화가 없다면 진행을 멈춘다.
    - 만약 계속 변화가 있다면 2~3 과정을 반복한다.

- K-평균 군집화 알고리즘은 다음 상황에서 데이터 분류가 원하는 결과와 다르게 발생할 수 있으므로 사용하지 않는 것이 좋다. 

**데이터가 비선형일 때**

<br/>

<img width="713" height="228" alt="image" src="https://github.com/user-attachments/assets/a705db24-e5d5-4153-a84d-860f062c9c92" />

<br/>

- 직선으로만 나눈다
    - k-평균은 두 중심점 사이의 수직 이등분선(직선)을 기준으로 데이터를 나눈다.
    - K=2 결과를 보면 초승달처럼 휘어진 데이터는 곡선으로 나누어야 하므로, 직선으로 자르는 순간 데이터가 엉뚱하게 쪼개진다.
- 동그란 모양만 찾는다.
    - 알고리즘 자체가 모든 데이터 덩어리가 동그란 공 모양이라고 가정하고 중심점을 잡는다.
    - 길쭉하거나 휘어진 모양은 억지로 동그라미 안에 넣을려다 보니 형태가 왜곡된다.
- 연결성보다 거리를 따진다
    - K-평균은 단순히 중심점에서 물리적으로 가까운지만 확인한다.
    - 휘어 있는 끝부분은 중심점보다 다른 팀 중심점에 더 가까울 수 있어 팀이 바뀌어 버리는 것이다.

**군집 크기가 다를 때**

<br/>

<img width="636" height="282" alt="image" src="https://github.com/user-attachments/assets/82b99ca1-ac08-4e49-9054-a2d70ac8d526" />

<br/>

- 평균의 함정
    - K-평균은 모든 군집이 비슷한 크기의 원형일 것이라고 가정한다.
    - 그래서 데이터가 중앙에 크게 뭉쳐있고 양옆에 작게 뭉쳐 있을 때, 중앙의 거대한 군집을 하나로 인식하지 못하고 쪼개서 옆의 작은 군집과 합쳐버린다.
- 가까운 중심점 우선 원칙
    - 알고리즘은 단순히 가장 가까운 중심점(+)에 데이터를 할당한다.
    - 중앙의 큰 군집에 속한 데이터라도, 물리적 거리가 오른쪽의 작은 군집 중심점과 더 가까우면 오른쪽 군집으로 강제 편입된다.
    - 결과적으로 그림처럼 큰 원의 일부가 잘려 나가고 양옆의 작은 군집들과 합쳐지는 잘못된 분류가 발생한다.
- 분산을 고려하지 않음
    - K-평균은 데이터가 얼마나 넓게 퍼져 있는지(분산)를 계산에 넣지 않는다.
    - 오로지 중심으로부터의 절대적인 거리만 보기 때문에, 넓게 퍼진 군집의 데이터를 좁게 모인 군집이 빼앗아 오는 현상이 일어난다.

**군집마다 밀집도(density)와 거리가 다를 때**

<br/>

<img width="657" height="291" alt="image" src="https://github.com/user-attachments/assets/344d76da-fffd-45e0-9057-75cd281f7b0e" />

<br/>

- 밀도를 무시한 거리 계산
    - K-평균은 데이터가 얼마나 빽빽하게 모여 있는지 상관없이 오직 중심점까지의 직선거리만 잰다.
    - 이미지 왼쪽은 데이터가 넓게 퍼져 있고, 오른쪽은 두 덩어리가 아주 빽빽하게 붙어있다.
    - K-평균은 오른쪽의 빽빽한 두 덩어리를 하나의 큰 덩어리로 묶으려고 하거나, 중심점이 애매하게 잡히면서 두 덩어리의 경계가 무너지는 결과를 초래한다.
- 전체 평균에 의한 왜곡
    - K-평균은 모든 데이터를 포함하는 평균 위치를 찾으려다 보니, 데이터가 성기게 퍼진 쪽은 중심점이 데이터를 다 포괄하지 못하고 쪼개지게 된다.
    - 반대로 밀도가 높은 쪽은 데이터가 조금만 떨어져 있어도 같은 군집으로 묶어버리는 경향이 생긴다.
- 거리의 불균형
    - 이미지를 보면 왼쪽 군집은 크고, 두 군집은 작고 서로 가깝다.
    - K-평균은 모든 군집의 크기(반지름)가 비슷하다고 가정한다.
    - 따라서 큰 군집의 일부 데이터를 억지로 떼어내어 옆에 있는 작은 군집의 중심점에 할당하는 영역 침범 현상이 발생한다.
