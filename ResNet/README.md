## ResNet
- ResNet 핵심은 깊어진 신경망을 효과적으로 학습하기 위한 방법으로 레지듀얼(residual) 개념을 고안한 것이다.
- 일반적으로 신경망 깊이가 깊어질수록 딥러닝 성능은 좋아질 것 같지만, 실상은 그렇지 않다.
- 논문에 따르면 신경망은 깊이가 깊어질수록 성능이 좋아지다가 일정한 단계에 다다르면 오히려 성능이 나빠진다고 한다. 

<br/>

<br/>

- 다음 그림과 같이 네트워크 56층이 20층보다 더 나쁜 성능을 보임을 알 수 있다.
- 즉, 네트워크 깊이가 깊다고 해서 무조건 성능이 좋아지지는 않는다는 것을 보여주고 있다.
- ResNet은 바로 이러한 문제를 해결하기 위해 레지듀얼 블록(residual block)을 도입했다.
- 레지듀얼 블록은 기울기가 잘 전파될 수 있도록 일종의 숏컷(shortcut, skip connection)을 만들어 준다.

<br/>

<img width="410" height="161" alt="image" src="https://github.com/user-attachments/assets/c7d52441-3025-4fe8-ba04-5c77de58325d" />

<br/>

- 이러한 개념이 필요한 이유는 2014년에 공개된 GoogLeNet은 층이 총 22개로 구성된 것에 비해 ResNet은 층이 총 152개로 구성되어 기울기 소멸 문제가 발생할 수 있기 때문이다.
- 따라서 다음 그림과 같이 숏컷을 두어 기울기 소멸 문제를 방지했다고 이해하면 된다.

<br/>

<img width="407" height="165" alt="image" src="https://github.com/user-attachments/assets/56113764-5b2f-4dbc-bd33-426881ff86f7" />

<br/>

- 블록은 계층의 묶음이다.
- 엄밀히 말해서 합성곱층을 하나의 블록으로 묶은 것이다.
- 색상별(보라색, 노란색 등)로 블록을 구분했는데 이렇게 묶인 계층들을 하나의 레지듀얼 블록이라고 한다.
- 그리고 레지듀얼 블록을 여러 개 쌓은 것을 ResNet이라고 한다.

<br/>

<img width="883" height="138" alt="image" src="https://github.com/user-attachments/assets/dc80ebef-9ac5-4f74-84fb-3f2736e7bb1c" />

<br/>

- 하지만 이렇게 계층을 계속해서 쌓아 늘리면 파라미터 수가 문제가 된다.
- 계층이 깊어질수록 파라미터는 증가한다. 이와 같이 계층의 깊이가 깊어질수록 파라미터는 무제한으로 커질 것이다.
- 이러한 문제를 해결하기 위해 병목 블록이라는 것을 두었다.

<br/>

<br/>

- 다음 그림은 ResNet34와 ResNet50이다.
- ResNet34는 기본 블록을 사용하며, ResNet50은 병목 블록을 사용한다.
- 기본 블록의 경우 파라미터 수가 39.3216M인 반면, 병목 블록의 경우 파라미터 수가 6.9632M이다.
- 깊이가 깊어졌음에도 파라미터 수는 감소한 것이다.

<br/>

<img width="294" height="165" alt="image" src="https://github.com/user-attachments/assets/4ec95f93-6c3c-45fb-ac16-552021d501e9" />

<br/>

- 합성곱층을 자세히 보면 ResNet34와 다르게 ResNet50에서는 3 x 3 합성곱층 앞뒤로 1 x 1 합성곱층이 붙어 있는데, 1 x 1 합성곱층의 채널 수를 조절하면서 차원을 줄였다 늘리는 것이 가능하기 때문에 파라미터 수를 줄일 수 있었던 것이다.
- 그리고 이 부분이 병목과 같다고 하여 병목 블록이라고 한다. 

<br/>

<br/>

- 이제 중요한 아이덴티티 매핑(혹은 숏컷(shortcut,), 스킵 연결(skip connection)이라고도 함)에 알아보자
- 아래 쪽에 + 기호가 있다.(기본 블록과 병목 블록 모두에서 사용된다.)
- 이 부분을 아이덴티티 매핑이라고 한다.
- 아이덴티티 매핑이란 입력 $x$가 어떤 함수를 통과하더라도 다시 $x$라는 형태로 출력되도록 한다.

<br/>

<img width="395" height="342" alt="image" src="https://github.com/user-attachments/assets/ed695a57-72c7-4901-922b-9f3beae3f77d" />

<br/>

- 다운샘플은 특성 맵 크기를 줄이기 위한 것으로 풀링과 같은 역할을 한다고 이해하면 된다.

<br/>

<img width="189" height="186" alt="image" src="https://github.com/user-attachments/assets/c995ce61-534a-4538-b02c-eb586e79b808" />

<br/>

- 보라색 영역의 첫 번째 블록에서 특성 맵의 형상이 (28, 28, 64)였다면 세 번째 블록의 마지막 합성곱층을 통과하고 아이덴티티 매핑까지 왼료된 특성 맵의 형상도 (28, 28, 64)이다.
- 노란색 영역의 시작 지점에서는 채널 수가 128로 늘어났고, /2라는 것으로 보아 첫 번째 블록에서 합성곱층의 스트라이드가 2로 늘어나 (14, 14, 128)로 바뀐다는 것을 알 수 있다.
- 즉, 보라색과 노란색의 형태가 다른데 이들 간의 형태를 맞추지 않으면 아이덴티티 매핑을 할 수 없게 된다.
- 그래서 아이덴티티에 대해 다운샘플이 필요하다. 

<br/>

<br/>

- 참고로 입력과 출력의 형태를 같도록 맞추어 주기 위해서는 스트라이드 2를 가진 1 x 1 합성곱 계층을 하나 연결해 주면 된다.
- 이와 같이 입력과 출력의 차원이 같은 것을 아이덴티티 블록이라고 하며, 입력 및 출력 차원이 동일하지 않고 입력의 차원을 출력에 맞추어 변경해야 하는 것을 프로젝션 숏컷 혹은 합성곱 블록이라고 한다.
